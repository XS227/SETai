SETai Workflow Showcase

See how SETai helps you propose, review, and deploy website changes with AI assistance.

1. Describe your update

Provide a plain-language request describing the change you want on your website. SETai parses the intent and prepares an actionable plan.

Input

"Add a hero banner with a call-to-action linking to pricing, and replace the testimonials section with a carousel."

2. Generate a proposal

SETai produces a proposed code diff, previewing HTML, CSS, and configuration changes. You can inspect the diff without touching your repo.

AI draft
  • New hero section with primary CTA
  • Updated navigation to include Pricing
  • Replaced testimonials list with carousel component

3. Review the diff

Compare code changes side by side, add comments, and iterate until the proposal meets your standards.

Quality gate

Before

<section class="testimonials">
-  <ul>
-    <li>"Great experience!"</li>
-  </ul>
</section>

After

<section class="testimonials">
  <div class="carousel">
    <button aria-label="Previous">◀</button>
    <div class="slides">...</div>
    <button aria-label="Next">▶</button>
  </div>
</section>

4. Approve and deploy

Click Approve to merge and deploy. SETai handles GitHub PRs, CI, and production rollout for you.

Deployment

Automated checks run before merge, then your live site updates with the approved changes.

5. Track history

View a chronological history of proposals, approvals, and deployments. Quickly roll back or revisit previous versions.
